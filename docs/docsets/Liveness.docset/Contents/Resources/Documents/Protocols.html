Protocols

The following protocols are available globally.

  • LivenessCameraDelegate

    LivenessCameraDelegate: A protocol that outlines the essential methods for receiving notifications from a liveness detection camera system within an iOS application. This delegate is crucial for integrating face detection functionalities with the UI and other components of the app, facilitating real-time interactions based on camera events. Implementers of this protocol can manage and respond to various camera-related events, such as face capture, face detection, and user movement.

    Example Implementations:

    • In a security application, an object conforming to LivenessCameraDelegate might use these callbacks to trigger additional security checks or alerts when a face is detected or when the detection system perceives abnormal head movements.
    • In a user verification flow, the delegate methods can help confirm user identity by analyzing how the user follows visual prompts (e.g., turning the head left or right as instructed by the UI).

    Responsibilities:

    • Managing the focus region on the camera to ensure optimal face detection.
    • Handling the capture of face images to process them for liveness verification.
    • Receiving and acting upon messages from the liveness detection system that might include success, failure, or further instructions.
    • Monitoring and responding to the degree of head movement as part of the liveness detection protocol, providing feedback or adjustments in the user interface as necessary.

    The delegate’s methods provide a structured way to interface with a camera controller that handles the complexities of video capture and face detection, ensuring that the application remains responsive and accurate in real-time scenarios.

  • LivenessDataSource

    LivenessDataSource A protocol in iOS development that defines the interface for supplying textual information related to various stages and alerts within a liveness detection system. This protocol plays a crucial role in localizing and customizing the user interface feedback during liveness detection processes, allowing dynamic content changes based on the specific action or alert encountered.

    Responsibilities:

    • Provides text that describes the current liveness alert, enhancing user understanding and interaction during the detection process.
    • Offers descriptive text for each liveness action to guide the user through the required actions during the liveness check.
    • Supplies detailed descriptions for each action to assist in clarifying what the user is expected to do.

    Example Usage:

    • In an application with multilingual support, implementing this protocol can dynamically provide text in the user’s preferred language, ensuring that the liveness detection process is accessible and understandable to all users.
    • In user experience-focused applications, the protocol can be used to provide detailed, context-specific guidance and feedback that enhances user engagement and compliance during security checks.

    Implementation:

    • liveness(textForAlert:): Method to return a string representing the message for specific alerts like face not found or multiple faces detected.
    • liveness(textForAction:): Method that provides instructional text for a given action, such as “Turn your head to the left”.
    • liveness(descriptionTextForAction:): Offers a more detailed explanation of the action, helping to ensure that the user is clear on what is expected.
    • liveness(textForPassedAction:): Delivers feedback once an action is successfully completed, such as “Action successfully completed”.

    This protocol is essential for making the liveness detection features in applications not only functional but also user-friendly and adaptable to various user environments and requirements.

  • LivenessDelegate

    LivenessDelegate A protocol in iOS development that defines the delegate methods used by LivenessCameraController to communicate liveness detection results back to the implementing class. This delegate protocol is essential for handling real-time feedback from the liveness detection system, including face detection, action progression, and video recording outcomes.

    Responsibilities:

    • Responds to successful face detection events by receiving an image of the detected face.
    • Receives notifications before an action is attempted, allowing the delegate to prepare or adjust the UI accordingly.
    • Handles the completion of actions, providing an opportunity to process results and determine the next steps.
    • Manages the handling of video recording outcomes, useful in scenarios where the liveness detection involves recording clips.
    • Acknowledges the successful completion of all liveness detection actions, signaling that the user has passed all required checks.

    Example Usage:

    • In a secure authentication flow, the delegate methods can be used to update the UI with instructions or feedback based on the user’s interaction with the liveness detection system.
    • In applications requiring enhanced security, such as banking apps, the delegate can trigger additional security measures or proceed with transaction authorization upon successful completion of liveness checks.

    Implementation:

    • The liveness(didCaptureFaceIn:) method can be implemented to store or analyze the captured face image.
    • liveness(willPassAction:) and liveness(didPassActionWith:) methods manage the flow of actions during liveness detection, ensuring that each step is logged or audited.
    • liveness(didRecordVideoTo:) provides a handle to the recorded video, potentially for review or archival.
    • livenessDidSucceed() confirms that all prescribed actions have been successfully completed, often used to unlock the next phase of an authentication or verification process.

    This protocol allows for tight integration between the camera controller and the broader application, ensuring that all aspects of liveness detection are appropriately managed and reflected in the user interface or system state.
